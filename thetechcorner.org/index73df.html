Unify controller is  a wireless network management software solution from Ubiquiti Networks. It allows you to manage multiple wireless networks using a web browser. The Sample Network Diagram above illustrates an example of a single UniFi network.

It’s a very useful piece of software, but it needs an external device to be working. You could use your main PC as the host, but I don’t know about you, but I for sure am not running my main PC 24/7. To fix this we can use Raspberry Pi, which is maybe already running PI-hole or NFS and so it won’t make difference on your electric bill.

1. Set-up Raspberry PI

I have already created a tutorial for this step, which you can check out here. You can configure PI with or without a monitor.

Next, you can check out some other guides on how to change your hostname and password or how you can overclock your Raspberry PI for better performance.

If you have already set-up your Raspberry, you only need to update your Raspbian packages and version using these commands

sudo apt update
sudo apt upgrade

2. Install necessary packages

As our next step, we need to install a few things to be to run Unify controller.

The first one is Java runtime environment 8, we cannot go with any newer version yet, because the UniFi controller doesn’t support it.

sudo apt install openjdk-8-jre-headless

To improve UniFi controller startup speed, we can install rng-tools which is a set of utilities related to random number generation in kernel. The main program is rngd, a daemon developed to check and feed random data from hardware device to kernel entropy pool. This is mainly useful to increase the quantity of entropy in kernel to make /dev/random faster

sudo apt install rng-tools

But it won’t work just yet, first we need to uncomment the HRNGDEVICE=/dev/hwrng line to define input source for random data.

sudo vi /etc/default/rng-tools-debian

And find #HRNGDEVICE=/dev/hwrng and remove the hastag.

Save the file and restart the rng-tools service

sudo systemctl restart rng-tools

3. Fixing the mangodb dependency issue

As the last thing of our preparation to install the UniFi Controller, we need to fix mangodb dependency issue, which you would get when trying to install it.

To fix this, we will create a workaround where we will change the package manager preference. We can do that in /etc/apt/preferences.d directory by adding a few lines into the mangodb file

sudo vi /etc/apt/preferences.d/99stretch-mongodb.pref

Now add these lines to the file

#Never Prefer packages from Stretch

Package: *

Pin: release n=stretch

Pin-Priority: 1

Save the file, and now it’s time to add our repository to /etc/apt/sources.list

echo "deb http://archive.raspbian.org/raspbian stretch main" | sudo tee /etc/apt/sources.list.d/stretch_mongodb.list

When this is done, let’s move the the last step of installation.

4. Install the UniFi Controller

We have a few options now, we can just manually download the UniFi controller and install it or we could automate the whole process with package manager and in that way, every time you run apt update, the UniFi controller updates as well.

To do that type

echo 'deb [signed-by=/usr/share/keyrings/ubiquiti-archive-keyring.gpg] https://www.ui.com/downloads/unifi/debian stable ubiquiti' | sudo tee /etc/apt/sources.list.d/100-ubnt-unifi.list

And add the GPG key 

curl https://dl.ui.com/unifi/unifi-repo.gpg | sudo tee /usr/share/keyrings/ubiquiti-archive-keyring.gpg >/dev/null

Finally, update the repositories and install UniFi controller

sudo apt update
sudo apt install unifi

This will automatically start and enable the unifi service. We can check that out by typing

systemctl status unifi

5. Setup the UniFi controller

We could now access the initial setup and the whole interface by typing the following line to our browser on the PI 

https://local.host:8443

But I would like to be able to access this interface from my own PC/phone. To do that, we need to check and set static IP address on the PI.

To check your IP type

ip a

As you can see mine is 192.168.100.30. To do this properly, you should link this IP with your device in your main router, but if you can’t access it, the best you can do is to set the static IP into your PI.

Everything should be now up and ready. All you have to do is type the following line into your browser on main PC.

https://YOUR_PI_IP:8443/
]]> 1459 Raspberry PI – Automatically Controlled Fan https://thetechcorner.org/?p=556&utm_source=rss&utm_medium=rss&utm_campaign=raspberry-pi-automatically-controlled-fan Fri, 17 Dec 2021 17:49:10 +0000 https://thetechcorner.org/?p=556 With the new Raspberry Pi 4 and it’s CPU, Raspberry tends to get really hot. Adding a fan can significantly help this problem, but there is going to be an issue with the annoying buzzing sound of the small fan.

In this guide, I’ll show you how to set up this fan, so it’s only starting working when the PI will get hot.

1. First, you will need a few things:

  1. Raspberry Pi (any model will do for this)
  2. SD card / USB
  3. Power supply
  4. Ethernet Cable (you could also you a wifi, but Ethernet is just way better in performance and reliability
  5. Small fan – 5V
  6. Few cables and soldering iron
  7. NPN transistor (2N2222) – 680Ω

2. Seting up your your PI.

You can find guide for that here!

3. Connecting the circuit

In this step, I’ll explain the circuit and a step-by-step guide to connecting it right.

Because the fan requirement of the 5V and the Pi capability of providing 3.3V at max, we added a NPN transistor. We could just use dedicated 5V pins that are on the PI, but they don’t have regulation capability build into them. That’s why we are going to use GPIO pins.

The transistor is an electrical component used in circuits to switch or amplify power. In our case, it’s going to be switch.

We are going to connect the positive lead aka Emitor (E )to 5v+ on the PI and the collector (C ) to the ground on the PI. Last but not least, we are going to connect the base (B ) of the transistor to the PIN 17 and place a 680 Ohm resistor between them. You can see which leg of the NPN transistor is what on the picture below as well as the GPIO name scheme of the PI

Finally, the final wired up product should look like this.

Here is my PI with the hot glue used to keep everything together and protected. It doesn’t look the nicest, but it gets the job done, and it’s pretty reliable.

4. Script for a fan

First, we will need to install a python 3 for our script to work.

sudo apt install python3-pip

Next, we will be using a script that some gentleman already wrote for us. Just copy and paste this command into your PI

git clone https://github.com/Howchoo/pi-fan-controller.git

We will need to install a few more packages, this process is already automated, so you just have to type

sudo pip3 install -r pi-fan-controller/requirements.txt

And finally start the script, that will automatically configure everything

./pi-fan-controller/script/install

Everything should be working now. To test this, we will use a stress test and measure the temperature along with it. In one terminal type:

sudo apt-get install stress
sudo stress --cpu 8 --timeout 60

And in the other window terminal we will measure the temperature and watch fans if they spin on in time:

watch -n 1 vcgencmd measure_temp

If everything goes correctly, a fan should turn on when temperature hits the 65° C

You can change the threshold of the fan settings by opening fancontrol.py config file and rewriting the temperatures: 

sudo vi /usr/local/bin/fancontrol.py

Your Pi should now be able to automatically control the fan, so both of you can be cool now.

]]> 556 How to recover a root password in a Linux https://thetechcorner.org/?p=503&utm_source=rss&utm_medium=rss&utm_campaign=how-to-recover-a-root-password-in-a-linux Thu, 25 Nov 2021 12:10:51 +0000 https://thetechcorner.org/?p=503 In case of losing a password of your Linux operating system, you can still recover it quite easily, as long as you have physical access to the system.

There are multiple options how to do this, but I will show two most often and the special one used in the Raspbian systems.

1. Reset root password from the GRUB

First, we need to get into the GRUB (bootloader that loads the Linux kernel or any other OS). To do that, we need to reboot our OS (this works with VM’s as well) and when you see Oracle’s logo, click the screen and hold down SHIFT until the Grub menu appears.

There is also the option to have this show with every boot. To do this you need to edit a grub file which you may not have access to so I’m just giving you this as a useful option in the future.

gksudo gedit /etc/default/grub

and add a # in front of GRUB_HIDDEN_TIMEOUT=0

And apply the settings with 

sudo update-grub

Choose an Advanced options for your OS and then press letter ‘e’ on your keyboard for edit option.

Now we need to rewrite some of the lines to tell the system to start with bash when system boots and to do that change system from read-only to read and write option.

So we just change rd -> rw and add init=/bin/bash

Press F10 to boot and display a screen with a command line.

Now we need to mount our root file system in read-write mode. To do that type:

mount -n -o remount, rw /

And we are free to change a password:

passwd root

Take a note, you can do this for any user account on this OS, not only root

To finish and reboot a system type:

exec /sbin/init

Reset root password with live CD

Another very popular option is to use live CD or USB, if you don’t have one simply download for example Ubuntu and burn in onto USB using Rufus or any other 3rd party app.

  1. In the first step simply put USB into a PC.
  2. Start PC and wait for a screen with two options, Try Ubuntu and Install Ubuntu. (note that you’ll maybe need to change a settings in BIOS to boot into live CD successfully) and pick the Try Ubuntu option.

3. Open a terminal and type following commands. This makes us a root user and will list disk partions:

sudo su 
fdisk -l

We are looking for a system partition, which in MY CASE is /dev/sd5 and is type Linux. This may differ in your case.

4. Now we need to mount this system to a folder which we will create:

mkdir /mnt/recovery
mount /dev/sda1 /mnt/recovery

5. In this step we need to trick the system into thinking we are using regular Linux file system. To do this we will “jail ourself” or technically speaking “chroot ourself” into this directory.

chroot /mnt/recovery

5. System now thinks we are root in regular Linux session and we can easily change the password of root or any user like in previous case with the GRUB method

passwd root
exit
umount /mnt/recovery

6. Remove live CD or USB and reboot the system

Reset root password in a Raspbian

This method is rather different from all the others so I decided to put this here as well.

  1. First you need to extract the SD card from your Raspberry PI and put it into your other PC with card reader.
  2. Open the SD card and edit the cmdline.txt file.
  3. You will see line like this
dwc_otg.lpm_enable=0 console=ttyAMA0,115200 console=tty1 root=/dev/mmcblk0p2 rootfstype=ext4 elevator=deadline fsck.repair=yes rootwait

And you simply add the line bellow to the end of the line above. Note this line MUST BE on the same line.

init=/bin/sh

4. Now put the SD card again to the Raspberry PI and boot it. The rest of this tutorial is the same as the first example with the GRUB so I’ll just speed run this

mount -o remount, rw /
password pi
sync
exec /sbin/init

]]> 499 How to create a NAS server on a Raspberry PI https://thetechcorner.org/?p=491&utm_source=rss&utm_medium=rss&utm_campaign=how-create-nas-server-using-raspberry-pi Fri, 26 Feb 2021 16:50:45 +0000 https://thetechcorner.org/?p=467 NAS stands for Network Attached Storage.

The NAS serves as a storage container for a files of all kinds, photos, videos, music, or movies. Its biggest advantage is that anyone on your local network (to whom you give access) can connect to it, so these files can be access from more than one computer.

For example, you can automatically back up photos from your phone when you come home.It’s very easy to build someting like that in a few minutes and all you need is some old PC or in our case Raspberry PI with external HDD on which we will store this data.

Select software

Technology is constantly evolving and the demand for external repositories is increasing. Therefore, there are also several different software solutions for creating NAS server.

FTP – File Transfer Protocol, one of the first protocols developed back in the 1970s

SAMBA – one of the most popular software for creating NAS servers mainly for Windows systems

NFS – Network Attached Storage originally created by Sun Microsystem in 1984

AFP – The Apple File protocol is similar to SAMBA but already as the name suggests it is mainly aimed at Apple systems and its products

In the future, I will make a tutorial on how to install each one of these on Raspberry PI, but SAMBA and its benefits outweigh all other software so I will use it for the time being.

What we’re going to need

  • Raspberry PI
  • SD card
  • Power cord
  • Ethernet cable (wifi can also be used, but cable is always faster and more reliable)
  • External HDD/SSD/USB

1. Set-up Raspberry PI

I have already created a tutorial for this step, which you can check out here. You can configure PI with or without a monitor.

Next, you can check out some other guides on how to change your hostname and password or how you can overclock your Raspberry PI for better performance.

If you have already set-up your Raspberry, you only need to update your Raspbian packages and version using these commands

sudo apt update
sudo apt upgrade

2. Configure NAS SAMBA

a) First we have to download SAMBA to our PI

sudo apt-get install samba samba-common-bin

After a few seconds/minutes, you should see the welcome screen

SAMBA will ask you for additional approval so that it can modify its settings based on dhcp server.

If you do not understand what this is press YES if you curius about how it works, you can click here.

b) If the installation was successful, then our next step will be to create a folder that we will share over the network. Because we want to use our server to store photos, movies, etc., this folder will have to be on an external HDD.

Tutorial on how to mount an external HDD on a Raspberry PI can be found here.

In my case, the HDD will be mounted to /mnt/HDD1

c) When we have a folder that we are going to share, we can start with the SAMBA software configuration.

Using the editor of your choice, we will open a SAMBA config in which we will write down the necessary data. I’ll use nano for this.

sudo nano /etc/samba/smb.conf

Scroll completely to the bottom of this text file and write/copy the following

[thetechcornershare]
path = /mnt/HDD1
writeable=Yes
create mask=0777
directory mask=0777
public=no

thetechcornershare is just a name that you can change to your likening

path defines the folder we want to share (in your case, this path may vary)

writeable=yes means that we want to be able to write this folder and not only to read files

create mask and directory mask determine what rights will user have. 0777 allows them to read, write, and execute files

public=no means that we want the user to enter a password to access the folder

Save the text file using shortcut ctrl + x and confirm the save using y and press ENTER.

d) As a next step, we need to create a new user to connect from different devices to our SAMBA NAS. This user must have the same name as one of the users on your PI. As most of us use the user “pi” so we will use this.

sudo smbpasswd -a pi

SAMBA will automatically ask us for a new password for this user.

I’ll leave the password option to you, I’ll use the “test” but you should choose something much more complex.

e) We will then restart the SAMBA service to add the new account to our system. We will do so by using the command

sudo systemctl reload smbd

f) As a last step, you will need to find out the hostname or the IP address of your Raspberry PI. The easiest way is to use the command 

hostname -I

This completes the first part of creating our new NAT server. Next, I’ll explain how to connect to NAS from your WIndows PC.

3. How to connect to a NAS from Windows

This part is quite simple. All you have to do is click on the “This PC” icon and click on the Computer box in the upper-left corner and choose the option to map the network drive

As the next step, we choose the name of the disk and define the path to our NAS shared folder.

As a disc name, you can select any letter except one that your PC already uses and we will write the following path in the Folder box

\\raspberrypi\thetechcornershare

Also, check both boxes for Reconnecting at singning in and connecting to your NAS with different credentials.

Keep in mind that if you chose a different name when configuring SAMBA, you must modify this value here.

Confirm by clicking the Finish button and Windows will then automatically ask you for login information for this NAS folder.

We will enter the data that we have predefined in the previous paragraph. The login name is “pi” and the password “test”.

Select Remember my credentials so you don’t have to enter this data every time you start your PC or Raspberry PI and press OK to confirm.

You should now see your NAS folder on your computer.

]]> 491 How to mount a USB drive/HDD on Raspberry PI https://thetechcorner.org/?p=461&utm_source=rss&utm_medium=rss&utm_campaign=as-mountnut-usb-disk-hdd-na-raspberry-pi https://thetechcorner.org/?p=461#comments Sat, 20 Feb 2021 09:30:39 +0000 https://thetechcorner.org/?p=448 If you plan to use your Raspberry PI as a medium to store photos, data, videos, or you planning on building a home PLEX server, then you’ll need your Raspberry PI to have as much storage as possible.

For example, an old USB, SSD, or HDD can help you with this. You can coonect them to your PI over the USB port.

If you’re using a 3.5 inch HDD that doesn’t have a USB plug and requires external power, you can use a reduction from SATA to USB, but make sure it includes this external power supply. There are several variations, but such a reduction should look something like this.

You will need

  • Raspberry PI
  • SD card
  • Ethernet cable (or you can just use WIFI)
  • External USB/HDD/SSD

1. Set-up Raspberry PI

I have already created a tutorial for this step, which you can check out here. You can configure PI with or without a monitor.

Next, you can check out some other guides on how to change your hostname and password or how you can overclock your Raspberry PI for better performance.

If you have already set-up your Raspberry, you only need to update your Raspbian packages and version using these commands

sudo apt update
sudo apt upgrade

2. Connect external HDDs to Raspberry PI

Once connected, the HDD should automatically be attached to Raspberry PI. We can check it with a command

df -h

You can you see my HDD is completely at the bottom of the list named /dev/sda1 (standard filesystem name for external disks) and it’s mouned on /media/pi/New Volume

If you don’t see your HDD in this list, try connecting it to a different USB port or rebooting your Raspberry Pi.

Raspberry PI automatically configured all the necessary parameters for us to mount the disk. We can find these parameters by typing “cating” the mounts where at the very bottom of the list we can see our /dev/sda1 disk

sudo cat /proc/mounts

3. Find your UUID and disk formatting type

To change some of these parameters, we will first need to detect the UUID and the type of formatting of our HDD. UUID stands for Universal Unique Identifier.

To find out this information, we will use the 

sudo blkid /dev/sda1

If your HDD has a different name assigned or you have multiple HDDs you will need to overwrite sda1 to sda2 etc…

Now we know that our HDD /dev/sda1 is an NTFS type with UUID “AAE87646E8761135”

Depending on the type of hdd you have, the following procedure will change.

4a. NTFS HDD packages

To use the NTFS format on your HDD, you will need to install an NTFS driver. To do this, use the

sudo apt install ntfs-3g

4b. ExFAT HDD Packages

You’ll need these two packages to use the exFAT format on your HDD.

sudo apt install exfat-fuse 
sudo apt install exfat-utils

5. Manual mounting

After detecting UUID, and type of HDD formating and installing the necessary packages, we can finally start the mounting process itself.

First of all, we need to create a folder to which we will mount this HDD

sudo mkdir -p /mnt/HDD1

As a next step, we must grant this folder the necessary rights of access for users. If you are using a different user than pi, then of course you must change this value in the command to your user’s name, e.g. test:test

sudo chown -R pi:pi /mnt/HDD1

Now we need to modify the fstab file that controls mounting devices to Raspberry PI. With this command, we open a text editor to insert the necessary data into the file.

sudo nano /etc/fstab

We’ll add a line to the file that looks like this. Keep in mind that instead and [UUID] and [TYPE] you have to insert your parameters.

UUID[UUID]= /mnt/HDD1[TYPE] defaults,auto,users,rw,nofail,noatime 0 0

For me, this line will look like this.

UUID=AAE87646E8761135 /mnt/HDD1 ntfs defaults,auto,users,rw,nofail,noatime 0 0

When you have written this line to a file, save it using ctrl+x and confirm the save using y and ENTER.

Becaouse Raspberry PI automatically mounted our HDD before, we must first unmount it. 

sudo umount /dev/sda1

Then reconnected it with the new parameters entered into the fstab file using the

sudo mount -a

Congratulations your external HDD should be mounted to a Raspberry PI! It should stay mounted even after the reboot of the system.

Is it some sort of cooling solution in which you drill a hole into your Raspberry Pi? Or maybe it is some cooking recipe?!

The answer is no, it’s something way better in today’s digital age.

According to Wikipedia Pi-Hole is a Linux network-level advertisement and internet tracker blocking application which acts as a DNS sinkhole and optionally a DHCP server, intended for use on a private network.

Beautifully said, but maybe I will try to explain it a little bit more “user friendly”. I’ll start with an example.

Do you know that feeling when you are scrolling on your phone and you see an interesting article, so you click on it and there are either 15 ads waiting for you or one so huge that you cannot see even title of an article?

Yea, I hate that too. But there is a simple solution to this!

A Pi-Hole is a peace of code that runs on a Linux (primarily Raspbian) and with very few installation commands it allows you to clear your sides and videos from all the unwanted ads which can even take up a lot of internet bandwidth.

Now you are like “I just turn on my ad-block, I don’t need no Pi-Hole for that” but no, this is much better and here is why.

A Pi-Hole works at the network level layer which means It’s not filtering them out after your browser downloaded them, you actually wont even download them, which can save up a lot of bandwidth.

Another great advantage of the Pi-Hole is that It cooperates with the router so ALL of the devices on your network will be ads free! You cannot install ad-block and all of your mobile apps that are using ads nor you cannot install ad-block on your TV. But with this you only need one device connected to your router and you are done.

Just imagine how many ads you see daily, then multiply it by number of devices in your network. How much bandwidth will you save? A lot right?

How does it work?

Ok, so If you don’t know what a DNS is, basically it’s a naming system in which IP address gets their own domain (for example 192.0.2.44 = www.amazon.com). This feature is usally provided by billion dollars companies like google, amazon…

Your router is configured in the way that when you visit a site, that site makes a lot of requests for a lot of the stuff. And guess what? It uses a DNS for that (well, mostly, I’ll talk about that later)

Here is where the Pi-Hole comes. In your network, the Pi will act as a DNS server. But you can now insert a list for all of the DNS domains you don’t want to see, a blocklist. This allows the Pi to filter out any ad requests from sites, apps, etc. This list is over 100 000 lines long and it’s always updating by a people that hate ads as we do!

How to set it up?

1. First, you will need a few things:

  1. Raspberry Pi (any model will do for this)
  2. SD card / USB
  3. Power supply
  4. Ethernet Cable (you could also you a wifi, but ethernet is just way better in performance and reliability)

2. Seting up your your PI.

You can find guide for that here!

3. Download and install the Pi-Hole

wget -O basic-install.sh https://install.pi-hole.net
sudo bash basic-install.sh

You should see a welcome window looking like this.

Just press OK to begin the installation. Pi-hole will ask you for a donation, but it’s totally up to you if you want to donate or not, the program will still be fully functional. I think Pi-Hole is great and maybe if you will like it too you can consider donating something small.

In the next step Pi-Hole will explain to you why you need a static IP address and in the next section will ask you to make your current IP address static. If you don’t want to use any particular IP address, than you should choose this one (there is actually one more step before this, but they added it here for some reason).

Pi-Hole will ask you for the upstream DNS provider to which it will connect. You can use this tool as a benchmark for the fastest one for you, but It really won’t make a big difference, so you can just choose Google or CloudFlare if you don’t like google tracking you (not that any other companies won’t do it).

Pi-Hole is just a computer, so it doesn’t know on its own what domains should it block. We have people for that, who make lists of domains for that (blocklists). In this next step Pi-Hole will ask you what blocklists you want to use. For maximum efficiency, you should let it on the default (that means having all of them checked). Don’t get scared if you have more or less than I do. People make and merge new together all the time. You can download more of these blocklists, but more on that later.

In the next step Pi-Hole will ask you what protocols you want to use. If you don’t know anything about Ipv4/Ipv6 just leave it default.

And now Pi-Hole will finally ask you if you want to use this IP address as your static IP address, which means It won’t change after reboot. If you don’t know anything about DHCP and static/dynamic IPs just press yes.

In the last step Pi-Hole will warn you that even if you set up your static IP on your PI your router can still assign it to another device. But with modern routers this shouldn’t really happen. Press OK.

Then you will have an option to install the web admin interface. I really like the web interface and I think you will like it too!

This feature requires a web server to work. They are only giving you the option because maybe you have your own web server and you don’t want another one. So just leave it on and continue.
