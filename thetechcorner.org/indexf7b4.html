Installing Terminator

The first step would be installing in what’s my opinion the best terminal emulator. It uses appearance settings, hotkeys, text search, arrange terminals. IT has many interesting and good-looking themes, which is also very important because you’ll be looking at the thing for many many hours.

Depending on the distro use

sudo apt-get install terminator -y
sudo dnf install terminator -y

Installing Zsh

As a default, you are probably using bash which is is a rewrite of the old Bourne shell (sh) that shipped in UNIX.

Zsh is intended to be friendlier than bash, through better interaction. Some of its useful features are:

  • Programmable command line completion
  • Shared command history between running shell sessions
  • Spelling correction
  • Loadable modules
  • Interactive selection of files and folders
sudo dnf install zsh -y

To start using it type “zsh” into the terminal and you should see similar output to this

Let’s leave everything in default for now, so simply type 0.

Set Zsh as the default shell

To do that type

chsh -s $(which zsh)

And if you had any custom ~/.bashrc settings or aliaes dont forget to move them to ~/.zshrc

Installing Oh my Zsh

 Oh my zsh is an open source, community-driven framework for managing your Zsh configuration. It comes bundled with thousands of helpful functions, helpers, plugins, themes etc.

To install it we use simple script provided by their github

sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

Type yes and you are good to go.

Changing a theme, background, fonts etc.

Right click on the terminator and choose “Preferences”. Now to into the Colors>Foreground and Background and go into the drop down menu. My theme of choice is “Solarized dark” and again “Solarized” for the Palette.

For the fonts we are going to use a package called “Powerline-fonts”

Simple use package manager to download it, but from my experience with this package, different distros have different package names, so watch out.

sudo dnf install powerline powerline-fonts -y

Change theme to Agnoster

You need to change ZSH_THEME="robbyrussell" to ZSH_THEME="agnoster" in your .zshrc

Restart your terminal.

Plugins

The great feature of ZFS is that you can add many more features in form of plugins. I’m gonna show you a few of my favorite ones

Autosuggestions

Autosuggestion work using your zsh history. This greatly increases your typing speed. https://github.com/zsh-users/zsh-autosuggestions

git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions

Add the plugin to the list of plugins for Oh My Zsh to load (inside ~/.zshrc): plugins=(zsh-autosuggestions)

Syntax highlighting
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git
echo "source ${(q-)PWD}/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh" >> ${ZDOTDIR:-$HOME}/.zshrc

Remove hostname

This is not really a plugin, but I like the clean look of it, therefore we’ll remove this. Add this line to end of your ~/.zshrc

prompt_context() {}

]]> 1479 Setting up a log rotate utility in Linux https://thetechcorner.org/?p=525&utm_source=rss&utm_medium=rss&utm_campaign=setting-up-a-log-rotate-utility-in-linux Thu, 02 Dec 2021 12:22:10 +0000 https://thetechcorner.org/?p=525 Arguably, one of the most used Linux directories is /var/log. Most of the services are logging all of the info into the logs in this directory. Administrators are using these logs to determine problems when troubleshooting the system or service.

I’ll use Raspbian OS just because I use it in most of my guides, so why not again.

We can see some of the content of my /var/log directory below:

We can see that some of the files end with a .gz extension, some of them have log in the name and some of them are combinations. They have one in common, and that is that they are logs.

For the purpose of not filling up the whole directory space, they are eventually divided and can be gziped aka they are being rotated.

Phrase rotated comes from a utility named Logrotate which whole purpose is to make this job of dividing files into smaller ones and gziping them up much easier.

Installing Logrorate

Depending on your package manager there are different ways to install this utility. In my case I’ll use Raspbian which is Debian based and comes with APT package manager

sudo apt-get install logrotate

Configurating Logrotate

Logrotate is very useful and all-around tool in which we can configure the parameters of how the log are going to be rotated.

We can configure these settings by creating a text file inside the /etc/logrotate.d directory. To do that, we type: 

sudo nano /etc/logrotate.d/apache2.conf

And inside these file paste following content 

/var/log/apache2/* {
    weekly
    rotate 3
    size 10M
    compress
    delaycompress
}

There are much more options that can be configured, you can just google them or use man logrotate

Let’s get over each line and explain it’s purpose:

  • weekly : as the name suggest, it means that logrotate will rotate the logs every week
    • we can also use value : daily, monthly
  • rotate 3 : implies that only 3 of the logs are going to be kept, thus older files than 3 are going to be deleted
  • size=10 : sets the minimal size of the logs to be 10M
  • compress : implies we want the logs to be gziped or compress by any other method
  • delaycompress : implies that we want the recent logs uncompressed until the next rotation cycle

Let’s try out these settings by forcing logrotate to rotate logs now with this command:

logrotate -d /etc/logrotate.d/apache2.conf

Alright, let’s try adding some additional settings to better understand this:

 /var/log/apache2/* { 
    weekly
    create 0644 root root
    rotate 4
    size 5m
    datetext
    dateformat -%d%m%Y
    notifempty
    mail filip@thetechcorner.org  
}

As you can see we have changed a few things. The log will now not by rotated, it’s just going to be moved and renamed to time and date when it reaches a 5M size. The mail log file will get new permission, which we set to 0644 with a root owner. And finally, when the number of logs reaches 5, the oldest one will be sent to the configured email address.

Cron

When we install logrotate utility by default, logrotate will create a file named logrotate inside /etc/cron.daily which will be executed daily at 6:25 am. You can check this by listing a content of this folder.

]]> 525 How to recover a root password in a Linux https://thetechcorner.org/?p=503&utm_source=rss&utm_medium=rss&utm_campaign=how-to-recover-a-root-password-in-a-linux Thu, 25 Nov 2021 12:10:51 +0000 https://thetechcorner.org/?p=503 In case of losing a password of your Linux operating system, you can still recover it quite easily, as long as you have physical access to the system.

There are multiple options how to do this, but I will show two most often and the special one used in the Raspbian systems.

1. Reset root password from the GRUB

First, we need to get into the GRUB (bootloader that loads the Linux kernel or any other OS). To do that, we need to reboot our OS (this works with VM’s as well) and when you see Oracle’s logo, click the screen and hold down SHIFT until the Grub menu appears.

There is also the option to have this show with every boot. To do this you need to edit a grub file which you may not have access to so I’m just giving you this as a useful option in the future.

gksudo gedit /etc/default/grub

and add a # in front of GRUB_HIDDEN_TIMEOUT=0

And apply the settings with 

sudo update-grub

Choose an Advanced options for your OS and then press letter ‘e’ on your keyboard for edit option.

Now we need to rewrite some of the lines to tell the system to start with bash when system boots and to do that change system from read-only to read and write option.

So we just change rd -> rw and add init=/bin/bash

Press F10 to boot and display a screen with a command line.

Now we need to mount our root file system in read-write mode. To do that type:

mount -n -o remount, rw /

And we are free to change a password:

passwd root

Take a note, you can do this for any user account on this OS, not only root

To finish and reboot a system type:

exec /sbin/init

Reset root password with live CD

Another very popular option is to use live CD or USB, if you don’t have one simply download for example Ubuntu and burn in onto USB using Rufus or any other 3rd party app.

  1. In the first step simply put USB into a PC.
  2. Start PC and wait for a screen with two options, Try Ubuntu and Install Ubuntu. (note that you’ll maybe need to change a settings in BIOS to boot into live CD successfully) and pick the Try Ubuntu option.

3. Open a terminal and type following commands. This makes us a root user and will list disk partions:

sudo su 
fdisk -l

We are looking for a system partition, which in MY CASE is /dev/sd5 and is type Linux. This may differ in your case.

4. Now we need to mount this system to a folder which we will create:

mkdir /mnt/recovery
mount /dev/sda1 /mnt/recovery

5. In this step we need to trick the system into thinking we are using regular Linux file system. To do this we will “jail ourself” or technically speaking “chroot ourself” into this directory.

chroot /mnt/recovery

5. System now thinks we are root in regular Linux session and we can easily change the password of root or any user like in previous case with the GRUB method

passwd root
exit
umount /mnt/recovery

6. Remove live CD or USB and reboot the system

Reset root password in a Raspbian

This method is rather different from all the others so I decided to put this here as well.

  1. First you need to extract the SD card from your Raspberry PI and put it into your other PC with card reader.
  2. Open the SD card and edit the cmdline.txt file.
  3. You will see line like this
dwc_otg.lpm_enable=0 console=ttyAMA0,115200 console=tty1 root=/dev/mmcblk0p2 rootfstype=ext4 elevator=deadline fsck.repair=yes rootwait

And you simply add the line bellow to the end of the line above. Note this line MUST BE on the same line.

init=/bin/sh

4. Now put the SD card again to the Raspberry PI and boot it. The rest of this tutorial is the same as the first example with the GRUB so I’ll just speed run this

mount -o remount, rw /
password pi
sync
exec /sbin/init
