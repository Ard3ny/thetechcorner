Now it’s time to use this container for something practical. To do that you obviously need to know first few basic things like: starting, stopping and listing containers.

To demonstrate this management commands, let’s make an apache server and run a site.

First, let’s find apache server container with podman help

podman search httpd | less

Now pick one and pull it as I showed you in the previous post. Just in the case you forgot, you have to type

podman pull registry.access.redhat.com/rhscl/httpd-24-rhel7

Run a container

To run a container we will again use podman command with few new parameters that I’ll explain.

podman run -d --name httpd_test registry.access.redhat.com/rhscl/httpd-24-rhel

Stop a container

podman stop httpd_test

List running/non-running containers

You can either list all container or just the containers that are running.

To list just the running containers, we use podman ps command

podman ps

To list all the containers, you have to add -a flag to the command

podman ps -a

List images

podman images

In the next post, I’ll explain to you how to run a service, which finally lead us to real life examples of container world.
