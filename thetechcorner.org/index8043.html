OSI Model

Simply put OSI model is a sequence of steps for communication between devices which are described in 7 layer model called the OSI model. It was made so there would be strict protocol, which all devices must follow, so there wouldn’t be any confusion and for people to understand how it all works.

It looks something like this and it works both ways. That means it starts with a Senders application layer going threw all of the layers, ends with Physical layer and then is sent over the network to the physical layer of the Reciever and again going threw all the layers and ends with the Application layer.

Let’s start from the top I think that’s the easiest way to explain it.

Layer 7: Application Layer

Communication starts on the top with the Appliccation layer which is closest to the end user. This layer interacts with the software applications. There applications are outside of the OSI model. Application layer functions typically include identifying communication partners, determining resource availability and synchronizing communication.

Layer 6: Presentation Layer

The presentation layer establishes a bridge between application layer entities (which can all have different syntax and langue) and the network format (which need to stay the same).

Layer 5: Session Layer

The session layer main function is to establish, manage and terminate the connections between the local and remote application. Simply put, it controls the dialogues (connections) between computers.

The session layer ensures that the session stays open long enough to transfer all the data being exchanged, and then promptly closes the session in order to avoid wasting resources.

Layer 4: Transport Layer

The transport Layer is responsible for end-to-end communication between the two devices. It takes data from the session layer and breaks them up into chunks called segments before sending it to layer 3.

Layer 4 is also responsible for the flow control (optimal speed so it won’t be too slow or too fast)and error control (when it receives the data, it makes sure that they are complete and request a retransmission they aren’t).

Layer 3: Network Layer

The network layer is responsible for managing data transfer between two different networks. If the two devices are on the same network, then this layer is unnecessary.

The network layer breaks up segments from the transport layer into even smaller units called packets. Then these packets are sent to the receiver and reassembled there.

This layer is also responsible for finding the best physical path for the data to reach its destination.

Layer 2: The Data link Layer

The data link layer is very similar to the network layer except it is used when data is transferred between devices on the same network.

The data link layer takes packets from the network layer and breaks them into even smaller pieces called frames.

As network layer it’s also responsible for the flow control and error control.

Layer 1: The Physical Layer

The physical layer is responsible for the transmission and reception of unstructured raw data between devices. It converts the digital bits (frames) into analog electrical, radio (wifi) or optical signal.

Layer specifications define characteristics such as voltage levels, the timing of voltage changes, physical data rates, maximum transmission distances, channel access method and physical connectors, etc.

TCP/IP Model

The OSI Model I just explained to you is just a reference/logical model. It was designed to describe the functions of the communication system by dividing the communication procedure into smaller and simpler components. But TCP/IP model was designed and developed by the Department of Defense (DoD) in 1960s and is based on standard protocols. It stands for Transmission Control Protocol/Internet Protocol. The TCP/IP model is a concise version of the OSI model. It contains four layers, unlike seven layers in the OSI model. The layers are:

  1. Process/Application Layer
  2. Host-to-Host/Transport Layer
  3. Internet Layer
  4. Network Access/Link Layer

Network Access Layer

Network Access Layer is the combination of Data Link Layer and Physical Layer available in the OSI model. Physical Addressing is done in this layer, i.e.  MAC Address of source and destination is assigned to the data packets. Hence this layer is responsible for the physical transmission of data.

Internet Layer

The Internet layer is used to send an independent packet to a network to the destination. All the machines, Web servers, nodes that are attached to the TCP/IP network are assigned in this layer.

Transport layer

It allows the information to be delivered in an informative chart format from the source to the destination host without defects.

Application Layer

Application layer performs the tasks of the OSI model in three layers: Application, Presentation and Session Layer. It is required for the interaction between node and node and handles the requirements of the user interface. Protocols available in the Application Layer are TFTP, HTTP, Telnet, SSH, NTP, DNS, NFS, FTP, SNMP, DHCP, SMTP (which I will talk about later in other posts).

Now you should have the slightest idea of how the data is sent from device to device. But let’s talk about the devices now.

Network hub

A network hub, also called ethernet hub, repeater or just hub is a networking device for connecting multiple ethernet devices together and making them act as a single network. If you know USB hubs this is same thing, but for ethernet devices.

It has multiple ports that works both ways, they receive but they also send data. This feature is called I/O – Input/output.

The way it works it’s very simple. Singnal introduced at the input of any ports appears at the output of every port except one that send the singnal. That’s why it’s called repeater, it only repeats the signal.

Hubs work at the Layer 1 (physical layer) of the OSI model. So it’s working with the raw analog signal. From all the devices I’m gonna tell you about this one is the most simple one. You can try reading again about the OSI model if you got lost.

One the left you can see how it may look like and on the right side there is a CISCO (one of the biggest network organizations) symbol for a hub.

Switch

A switch is a networking device that also connects multiple devices together to the one network. That includes computers, printers and servers. Because of the switch they can all communicate together regardless of their position.

You may be asking yourself now, hmm but what’s the difference, isn’t this just the hub? The difference between a hub and a switch is that switch only forwards data to the device that it should receive but hub sends data to all of the devices.

For that switch uses MAC addresses to forward data to the correct destination.

It’s a Layer 2 device operating on the data link layer of the OSi model.

Router

A router is a networking device that serve two primary functions:

  • creating and maintaing a local area network
  • managing the data (inside the network and entering the networking as well)

Simply put they connect computer networks together. For example, one is connecting your home network to the internet just now. But don’t get confused the one that you have in your home is a multifunction device that serve as a Router, Wifi Access Point, Switch and even more.

Routers operate at Layer 3 (network layer) of the OSI model.

Ultimately, a router works as a dispatcher, directing traffic and choosing the most efficient route for information, in the form of data packets, to travel across a network. A router connects your device to the world, protects information from security threats, and even decides which devices have priority over others.

LAN

A local area network (LAN) is a computer network that interconnects computers within a limited area (your home, school, office building).

According to CISCO a LAN comprises cables, access points, switches, routers, and other components that enable devices to connect to internal servers, web servers, and other LANs via wide area networks (WAN).

The rise of virtualization has also fueled the development of virtual LANs, which enable network administrators to logically group network nodes and partition their networks without a need for major infrastructure changes.

For example, in an office with multiple departments, such as accounting, IT support, and administration, each department’s computer could be logically connected to the same switch but segmented to behave as if they are separate.

WAN

A wide area network (WAN) is a network that extends over a large geographic area.

It’s a collection of LANs, or other networks that communicate together.

Simply but It ties all of LANs together.
