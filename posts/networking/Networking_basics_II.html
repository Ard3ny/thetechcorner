Networking basics II – DNS, SSH, NFS, NAT

In the previous post I’ve explained to you the very basics of the networking, what’s the OSI Model, TCP/IP model, Router, Switch, LAN and WAN.

In the second chapter of this series, I’m going to continue with another important protocols that are regularly used in administration or even non-tech person life.

DNS

You could say that the Domain Name System or shortly DNS is the phone book of the internet.

So what is does is translation of the IP addresses to domain names.

While the IP convention scheme allows computers to send each other messages, it’s not really handy for people to remember. For example, what would be easier for you to remember, the domain www.google.com or 8.8.8.8.? Maybe that was a bad example because the IP of Google is so easy, but you get the point.

How does it work?

The process of DNS resolution involves converting a hostname (www.test.com) into a computer friendly IP address, such as 192.168.1.1. An IP address is given to each device on the Internet, and that address is necessary to find the appropriate Internet device – like a street address is used to find a particular home. When a user wants to load a webpage, a translation must occur between what a user types into their web browser (example.com) and the machine-friendly address necessary to locate the example.com webpage.

SSH

SSH or Secure Shell is a network communication protocol that enables two computers to communicate with each other and share data. This communication is encrypted, meaning it’s suitable for use on an insecure network.

It’s most used case scenario is login and remote work on computers/servers, but it’s also possible to transfer data with it.

NFS

Network File System or NFS is a protocol invented in the 80’s to facilitate better remote file sharing between computers. It enables the storage and retrieval of the data from multiple disks and directories across a shared network.

An NFS uses a basic system where a “mount” command will prompt the server to link with many clients. The clients will get access to the same files on the server through the proper platform. The design can use security protocols to dictate who will access certain files, producing a simplified and safe approach to work.

How does it work?

An NFS can also use a file locking system that allows many clients to share the same files. The NFS can manage multiple application or compute threads for operation.
The NFS uses several hosts that will access the same files. It does not require one application for its operation, providing a simple design for handling the content one will read.


NAT

Network Address Translation or NAT is a process in which one or more local IP address is translated into one or more global IP addresses.

The idea of NAT is to allow multiple devices to access the internet through a single public address.

How does it work?

Generally, the border router is configured for NAT i.e the router which has one interface in the local (inside) network and one interface in the global (outside) network. When a packet traverse outside the local (inside) network, then NAT converts that local (private) IP address to a global (public) IP address. When a packet enters the local network, the global (public) IP address is converted to a local (private) IP address. 

Firewall

A firewall is a security device, it can be computer hardware or software. Firewall can help protect your network by filtering traffic and blocking outsiders from gaining unauthorized access to the private data on your computer.

Not only does a firewall block unwanted traffic, it can also help block malicious software from infecting your computer.

A firewall acts as a gatekeeper. It monitors attempts to gain access to your operating system and blocks unwanted traffic or unrecognized sources.

How does it do this? A firewall acts as a barrier or filter between your computer and another network such as the internet. 

How does it work?

To start, a firewalled system analyzes network traffic based on rules. A firewall only welcomes those incoming connections that it has been configured to accept. It does this by allowing or blocking specific data packets — units of communication you send over digital networks — based on pre-established security rules.

A firewall works like a traffic guard at your computer’s entry point, or port. Only trusted sources, or IP addresses, are allowed in. IP addresses are important because they identify a computer or source, just like your postal address identifies where you live.
