Most of the services are logging all of the info into the logs in this directory. Administrators are using these logs to determine problems when troubleshooting the system or service.

I’ll use Raspbian OS just because I use it in most of my guides, so why not again.

We can see some of the content of my /var/log directory below:

We can see that some of the files end with a .gz extension, some of them have log in the name and some of them are combinations. They have one in common, and that is that they are logs.

For the purpose of not filling up the whole directory space, they are eventually divided and can be gziped aka they are being rotated.

Phrase rotated comes from a utility named Logrotate which whole purpose is to make this job of dividing files into smaller ones and gziping them up much easier.

Installing Logrorate

Depending on your package manager there are different ways to install this utility. In my case I’ll use Raspbian which is Debian based and comes with APT package manager

sudo apt-get install logrotate

Configurating Logrotate

Logrotate is very useful and all-around tool in which we can configure the parameters of how the log are going to be rotated.

We can configure these settings by creating a text file inside the /etc/logrotate.d directory. To do that, we type: 

sudo nano /etc/logrotate.d/apache2.conf

And inside these file paste following content 

/var/log/apache2/* {
    weekly
    rotate 3
    size 10M
    compress
    delaycompress
}

There are much more options that can be configured, you can just google them or use man logrotate

Let’s get over each line and explain it’s purpose:

  • weekly : as the name suggest, it means that logrotate will rotate the logs every week
    • we can also use value : daily, monthly
  • rotate 3 : implies that only 3 of the logs are going to be kept, thus older files than 3 are going to be deleted
  • size=10 : sets the minimal size of the logs to be 10M
  • compress : implies we want the logs to be gziped or compress by any other method
  • delaycompress : implies that we want the recent logs uncompressed until the next rotation cycle

Let’s try out these settings by forcing logrotate to rotate logs now with this command:

logrotate -d /etc/logrotate.d/apache2.conf

Alright, let’s try adding some additional settings to better understand this:

 /var/log/apache2/* { 
    weekly
    create 0644 root root
    rotate 4
    size 5m
    datetext
    dateformat -%d%m%Y
    notifempty
    mail filip@thetechcorner.org  
}

As you can see we have changed a few things. The log will now not by rotated, it’s just going to be moved and renamed to time and date when it reaches a 5M size. The mail log file will get new permission, which we set to 0644 with a root owner. And finally, when the number of logs reaches 5, the oldest one will be sent to the configured email address.

Cron

When we install logrotate utility by default, logrotate will create a file named logrotate inside /etc/cron.daily which will be executed daily at 6:25 am. You can check this by listing a content of this folder.

]]> 525 Set Up New Minecraft 1.17 Server on Rasppbery PI [Java16] https://thetechcorner.org/?p=499&utm_source=rss&utm_medium=rss&utm_campaign=set-up-new-minecraft-1-17-server-on-rasppbery-pi-java16 Mon, 22 Nov 2021 13:16:14 +0000 https://thetechcorner.org/?p=499 With the new Minecraft version 1.17 you have to now use Java 16, which is not a problem on most Linux kernels, but so far Raspbian doesn’t natively support this. So in this quick guide, I’ll show you how to upgrade or setup this server on a new Java version.

I had previously posted a guide about setting up a Minecraft server, you can check it out here. But now few things have changed, so the process is a little different.

What will you need?

  1. Raspberry PI 3 or 4 (I’ll be using Raspberry PI 4B 2GB) with a fan
  2. Power adapter
  3. LAN cable
  4. HDMI cable (optional)

You can use HDMI cable as an output for your monitor, but I made a guide how to set up Raspberry PI and only use a LAN cable and your PC and I think that’s way more convenient, so I’ll not by using HDMI. You can find guide for that here.

1. Setting up a Raspberry PI

I made an entire guide for that, so you can check it out here.

2. Overclocking a Rasppberry PI

Minecraft server requires a lot of CPU power, so for that we will overclock it a little. If you don’t know how, you can check out my other guide for that here, where I explained everything and guide you through the whole process.

3. Downloading and installing Java 16

  1. sudo apt install openjdk-8-jdk
  2. wget https://github.com/AdoptOpenJDK/openjdk16-binaries/releases/download/jdk-16.0.1%2B9/OpenJDK16U-jdk_arm_linux_hotspot_16.0.1_9.tar.gz
  3. sudo mv OpenJDK16U-jdk_arm_linux_hotspot_16.0.1_9.tar.gz /lib/jvm && cd /lib/jvm
  4. sudo tar xvf OpenJDK16U-jdk_arm_linux_hotspot_16.0.1_9.tar.gz && cd OpenJDK16U-jdk_arm_linux_hotspot_16.0.1_9
  5. sudo update-alternatives –install /usr/bin/java /usr/lib/jvm/jdk-16.0.1+9/bin/java 1131

And now we can check if the new version is installed with this command

sudo update-alternatives --config java

The rest of the guide stays the same, you can just follow it here.

]]> 499 How to Change Raspberry PI’s Hostname / Password https://thetechcorner.org/?p=177&utm_source=rss&utm_medium=rss&utm_campaign=how-to-change-raspberry-pis-hostname-password Wed, 06 Jan 2021 10:59:49 +0000 https://thetechcorner.org/?p=177 By default your Raspberry PI’s hostname is raspberrypi and your password is raspberry.

There is nothing wrong with this name until you have multiple PIes in your network and trust me when you buy one, you will want more of them to do even more cool projects.

So in this short guide I’ll show you how to change the hostname and the password.

We can choose multiple paths to do so. In this guide I’ll show you one way over CLI (console) and one way over GUI (graphical interface)

Changing Hostname / Password Using CLI

Because our PI is running Raspbian we can use one of it’s features that is called raspi-config which allow us to change these things with very friendly user interface.

Connect and login to your PI, you can find how in this guide.

In the console type:

sudo raspi-config

You should see option menu like this. Choose the System option.

Select the S4 Hostname and choose a new hostname. Keep in mind that you can only use ASCII letters a-z, the digits 0-9 and the hyphen, but hostname cannot begin nor end with hyphen.

Congratulation, now do the same for the password and you will be send into a terminal in which you will type your new password.

Keep in mind that you need to reboot your device before hostname changes will take a place. For reboot just type:

sudo shutdown -r +0

Changing Hostname / Password Using GUI

If you have installed Raspberry PI OS with desktop and you are more used to work with graphical intreface, this is even easier method of changing the hostname or password.

First you need to connect your monitor or launch VNC to connect to the PI with ahernet cable. You can find guide for that here.

In the left upper corner select Raspbian Icon and select Prefrences>Raspberry Pi Configuration.

You should see an option to change hostname and password. Click into the window where it says raspberrypi and rename it to whatever you want.

Then click on Change Password and pick your new password.

Congratulation, you have successfully changed your hostname and password. Now you dont have worry about logging into a wrong PI or about somebody logging into yours!

]]> 177 Setup your Raspberry PI with / without a monitor https://thetechcorner.org/?p=136&utm_source=rss&utm_medium=rss&utm_campaign=raspberri-pi-setup-with-or-without-monitor https://thetechcorner.org/?p=136#comments Tue, 05 Jan 2021 13:12:35 +0000 https://thetechcorner.org/?p=136 If you are reading this then you prabaly know what a Raspberry PI is, but if you don’t it’s ok I’ll try to explain it to you as simple as possible and guide you threw the whole setup process.

First, we need to clarify what Raspberi PI is:

The Raspberry Pi is a low cost, credit-card sized computer that plugs into a computer monitor or TV, and uses a standard keyboard and mouse. It is a capable little device that enables people of all ages to explore computing, and to learn how to program in languages like Scratch and Python. It’s capable of doing everything you’d expect a desktop computer to do, from browsing the internet and playing high-definition video, to making spreadsheets, word-processing, and playing games.

So bassically, the Raspberry Pi is just a small computer that you can program to do pretty much anything with the right sensors and code.

The instructions that you will find here will guide you threw whole process of installation and will concist of the following:

  1. All the necessary equipment you will need
  2. How to download and write the operating system onto SD card
  3. Installation process with a monitor
  4. Installation process without a monitor

All the necessary equipment you will need

  1. Raspberry PI (for this guide I will be using Raspberry PI 4 B, but you can use any model)
  2. Power supply for model you are using (you will need stronger power supply with newer model)
  3. SD Card or USB Flash Drive
  4. Network or HDMI cable depending if you want to use monitor or not (for model 4B you will need micro HDMI)
  5. PC or laptop for downloading the OS and writing the image one SD card / USB

How to download and write the operating system onto SD card

  1. First you will need to download some sort of operating system. There are probabaly at minimum around 20 OS’s you can run on Raspberry PI. You can even run the Windows ARM version on your PI. But then your full small programmable computer that can do pretty much anything kinda loses it’s purpose, right? Therefore, for this example we will run Raspbian, because it’s the easiest one to run and is probably the most compatible one as it was made exactly for this reason. But you can check out all the other possible OS’s here.

You can download three versions of Rasbian. Lite version, OS with desktop GUI version and OS with desktop GUI and all the other recommended software version.

We will continue with the Raspberry PI OS with desktop and recommended software because is the one that requries the least knowledge if you are not familiar with using a terminal or Linux. You can download the newest version here.

2. Our next step will be downloading the software that will help us with writing the image. My favourite one is Rufus so I’ll be using that. You will need to insert your SD card or USB into your PC and open up Rufus. Then, you will select the Device on which you want to write the image and SELECT the image that you want to write and then you press START.

Installation process with a monitor

After the proccess of writing the image is complete you will need to insert the SD card/USB into your Raspberry PI and connect your HDMI and power supply as well as your peripherals (keyboard and mouse). After a few seconds you should be greeted with a booting screen and a configuration process of your new OS in which you will select your country, timezone, language and the password of your wifi network. Now, If you connect your PI to WIFI, the system will ask you if you wish to automaticaly update your software packages to the newest version (If you don’t want to connect to WIFI now, or you want to use the ethernet cable, I will show you how to do so later on in this guide).

Installation process without a monitor

If you are like me and you had no idea that Raspberry PI 4 B comes with a mini HDMI port and not a normal sized version, this guide is for you!

The proccess is similar to an installation with a monitor, but first we need to add one more file into the booting section so Raspberry PI will know we want to connect to it over SSH. Now we need to create an empty file named “SSH”. The tricky part is that windows isn’t exactly the best thing for doing this easily. First, we need to right click>new text document and call it SSH. However, we just made a text file and we need an empty file right? To do this, we need to open our SSH.txt file and in the left upper corner click File>Save as and we will rename it to “SSH” with the quotation marks and select our Save as type as All Files. Then just press save and we should have an empty file called SSH.

After this you can insert the SD card/USB into a PI, connect all your peripherals (keyboard and mouse) and a network cable. The other side of the network cable connects to your PC.

We will need some sort of SSH client. One of the most popular and user friendly ones is the PuTTY. You can download it here. After you have downloaded and installed PuTTY, type raspberrypi.local into a hostname and select port 22. After that just select Open.

After that you will be prompted with the console asking for your login information. The default login and password information are

Login: pi

Password: raspberry

After this we need to enable the VNC interface which allows us to send display output over the network interface. We can do this by typing

 pi@raspberrypi:~ $ sudo raspi-config

You will see this menu in which you will select Interface Option and then P3 VNC.

Then go back into the menu again and select System Options and S5 Boot / Auto login and then B3 Desktop.

And now just leave the menu with ESC and for a reboot type 

sudo shutdown -r +0

After a few minutes your PI should be up again and we can continue.

We will need to download the VNC that serves as a virtual bridge to our PI display interface. Install and open the VNC and on the top side of the software type the hostname of our device which we previosly entered into the PuTTY is raspberrypi.local. Enter our username which is PI and password raspberry and press ok.

With that your setup of the Raspberry PI is completed and you can fully enjoy it as you would be able to with the connected display.

Final configuration of the Raspbian OS

After booting into your OS with or without a monitor you will need to make some finnal configurations of the system, such as updating your OS, changing your username, password, resolution etc. I will make another guide for this soon, but for now, have a nice day and enjoy your piece of PI.
