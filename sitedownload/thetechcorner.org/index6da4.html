You’ve probably done this task many times before on your PC or phone, but in Linux everything is done over command, so let’s get right into it.

List password information about the user

chage -l username

Change the password

As a simple user without the privileges you can only change your own password but as a root you can change every user password including the root.

passwd username

Adjust password aging

Set the maximum number of days during which a password is valid (90 days for example)

chage -M 90 username

Set the minimum number of days between password changes (30 days for example)

chage -m 30 username

To make user immediately change the password after login

chage -d 0 username

Set expire date

chage -E EXPIRE_DATE username
]]> 1271 How to recover a root password in a Linux https://thetechcorner.org/?p=503&utm_source=rss&utm_medium=rss&utm_campaign=how-to-recover-a-root-password-in-a-linux Thu, 25 Nov 2021 12:10:51 +0000 https://thetechcorner.org/?p=503 In case of losing a password of your Linux operating system, you can still recover it quite easily, as long as you have physical access to the system.

There are multiple options how to do this, but I will show two most often and the special one used in the Raspbian systems.

1. Reset root password from the GRUB

First, we need to get into the GRUB (bootloader that loads the Linux kernel or any other OS). To do that, we need to reboot our OS (this works with VM’s as well) and when you see Oracle’s logo, click the screen and hold down SHIFT until the Grub menu appears.

There is also the option to have this show with every boot. To do this you need to edit a grub file which you may not have access to so I’m just giving you this as a useful option in the future.

gksudo gedit /etc/default/grub

and add a # in front of GRUB_HIDDEN_TIMEOUT=0

And apply the settings with 

sudo update-grub

Choose an Advanced options for your OS and then press letter ‘e’ on your keyboard for edit option.

Now we need to rewrite some of the lines to tell the system to start with bash when system boots and to do that change system from read-only to read and write option.

So we just change rd -> rw and add init=/bin/bash

Press F10 to boot and display a screen with a command line.

Now we need to mount our root file system in read-write mode. To do that type:

mount -n -o remount, rw /

And we are free to change a password:

passwd root

Take a note, you can do this for any user account on this OS, not only root

To finish and reboot a system type:

exec /sbin/init

Reset root password with live CD

Another very popular option is to use live CD or USB, if you don’t have one simply download for example Ubuntu and burn in onto USB using Rufus or any other 3rd party app.

  1. In the first step simply put USB into a PC.
  2. Start PC and wait for a screen with two options, Try Ubuntu and Install Ubuntu. (note that you’ll maybe need to change a settings in BIOS to boot into live CD successfully) and pick the Try Ubuntu option.

3. Open a terminal and type following commands. This makes us a root user and will list disk partions:

sudo su 
fdisk -l

We are looking for a system partition, which in MY CASE is /dev/sd5 and is type Linux. This may differ in your case.

4. Now we need to mount this system to a folder which we will create:

mkdir /mnt/recovery
mount /dev/sda1 /mnt/recovery

5. In this step we need to trick the system into thinking we are using regular Linux file system. To do this we will “jail ourself” or technically speaking “chroot ourself” into this directory.

chroot /mnt/recovery

5. System now thinks we are root in regular Linux session and we can easily change the password of root or any user like in previous case with the GRUB method

passwd root
exit
umount /mnt/recovery

6. Remove live CD or USB and reboot the system

Reset root password in a Raspbian

This method is rather different from all the others so I decided to put this here as well.

  1. First you need to extract the SD card from your Raspberry PI and put it into your other PC with card reader.
  2. Open the SD card and edit the cmdline.txt file.
  3. You will see line like this
dwc_otg.lpm_enable=0 console=ttyAMA0,115200 console=tty1 root=/dev/mmcblk0p2 rootfstype=ext4 elevator=deadline fsck.repair=yes rootwait

And you simply add the line bellow to the end of the line above. Note this line MUST BE on the same line.

init=/bin/sh

4. Now put the SD card again to the Raspberry PI and boot it. The rest of this tutorial is the same as the first example with the GRUB so I’ll just speed run this

mount -o remount, rw /
password pi
sync
exec /sbin/init

]]> 503 How to Change Raspberry PI’s Hostname / Password https://thetechcorner.org/?p=177&utm_source=rss&utm_medium=rss&utm_campaign=how-to-change-raspberry-pis-hostname-password Wed, 06 Jan 2021 10:59:49 +0000 https://thetechcorner.org/?p=177 By default your Raspberry PI’s hostname is raspberrypi and your password is raspberry.

There is nothing wrong with this name until you have multiple PIes in your network and trust me when you buy one, you will want more of them to do even more cool projects.

So in this short guide I’ll show you how to change the hostname and the password.

We can choose multiple paths to do so. In this guide I’ll show you one way over CLI (console) and one way over GUI (graphical interface)

Changing Hostname / Password Using CLI

Because our PI is running Raspbian we can use one of it’s features that is called raspi-config which allow us to change these things with very friendly user interface.

Connect and login to your PI, you can find how in this guide.

In the console type:

sudo raspi-config

You should see option menu like this. Choose the System option.

Select the S4 Hostname and choose a new hostname. Keep in mind that you can only use ASCII letters a-z, the digits 0-9 and the hyphen, but hostname cannot begin nor end with hyphen.

Congratulation, now do the same for the password and you will be send into a terminal in which you will type your new password.

Keep in mind that you need to reboot your device before hostname changes will take a place. For reboot just type:

sudo shutdown -r +0

Changing Hostname / Password Using GUI

If you have installed Raspberry PI OS with desktop and you are more used to work with graphical intreface, this is even easier method of changing the hostname or password.

First you need to connect your monitor or launch VNC to connect to the PI with ahernet cable. You can find guide for that here.

In the left upper corner select Raspbian Icon and select Prefrences>Raspberry Pi Configuration.

You should see an option to change hostname and password. Click into the window where it says raspberrypi and rename it to whatever you want.

Then click on Change Password and pick your new password.

Congratulation, you have successfully changed your hostname and password. Now you dont have worry about logging into a wrong PI or about somebody logging into yours!
