I found that for me the best type of relax is playing a game with a friends. I may sound childish, but I think minecraft is one of the best ones for relaxing.

You don’t need to try hard or think hard, you don’t have to have amazing reflexes or best strategic mind. It’s a game for everybody, but It’s still a lot of fun, especially when there are more of you.

But for that you need a server to let others join your game. Sure, you can do that from your desktop computer or notebook, but what happen when you will leave? They all have to leave and it’s get pretty frustrating when you can only play when a certain someone is online.

So another option is to rent a server from hosting platforms. But this can get pretty expensive (depending on hosting platform and a viable cpu/ram).

But luckily there is another option. You can host a server on your old PC or notebook or you can use a Raspberry PI as I’ll show in this guide.

What will you need?

  1. Raspberry PI 3 or 4 (I’ll be using Raspberry PI 4B 2GB) with a fan
  2. Power adapter
  3. LAN cable
  4. HDMI cable (optional)

You can use HDMI cable as an output for your monitor, but I made a guide how to setup Raspberry PI and only use a LAN cable and your PC and I think that’s way more convenient, so I’ll not by using HDMI. You can find guide for that here.

1. Setting up a Raspberry PI

I made and entire guide for that, so you can check it out here.

2. Overclocking a Rasppberry PI

Minecraft server requires a lot of CPU power, so for that we will overclock it a little. If you don’t know how, you can check out my other guide for that here, where I explained everything and guide you threw the whole proccess.

3. Downloading & Installing a Minecraft Server

a) First, we need to check if your PI has Java and Java complier. You can check that out by typing following commands into a terminal:

java  -version
javac â€“version

You should see something like this. Don’t worry if you have different versions, these packages are frequently updated and optimized.

b) First, we need to create a new repository to store a server file. We can create one in the home folder to make it simple. For that just type following into the terminal:

cd /home/pi
mkdir minecraft_server
cd minecraft_server

Next we need to download one of the minecraft servers. There are many options, but if you want something basic but still wants to have an option to install some plugins the Spigot Minecraft Server is a best choice.

To be precise you don’t really download a whole server, you download a tool that will build it. We will do that with a following command:

sudo wget https://hub.spigotmc.org/jenkins/job/BuildTools/lastSuccessfulBuild/artifact/target/BuildTools.jar

c) In the next step we need to build a server from a file that we have downloaded. This takes some time depending on your CPU. But if you are using Raspberry PI 4 and you even overclocked it as I did, it’ll be done in like 10 minutes, but don’t worry if it takes longer, it can take as long as one hour.

To start building a server type following. In the meantime, go grab a coffee or piece of Pi!

sudo java -jar BuildTools.jar

After 10-60 minutes you should see a Success message in your terminal.

4. Launching The Server

After we have successfully downloaded and build a server we can finally launch it! DEPENDING ON YOUR VERSION you will type following

sudo java -Xms512M -Xmx1024M -jar /home/pi/minecraft_server/spigot-YourVersion.jar nogui

But instead of “YourVersion” you need to type in your version of the server. For example, you can find mine in an image above. It’s “spigot-1.16.5.jar” so my command will look like this:

sudo java -Xms512M -Xmx1024M -jar /home/pi/minecraft_server/spigot-1.16.5.jar nogui

If you can’t find your just type 

ls | grep spigot

And you should see something like this

Also, it should be noted that the -Xmx and -Xms flags are allocating the memory used for the server. If you want to increase these values, you can just change the values in this command (in multiples of 1024). (But for 2 GB file this is the best option, you can maybe push it to 1.5GB, I’m also running a lot of other things on my PI so I only allocated 1 GB)

After this you should see an error message asking you for agreeing with end-user licence agreement (EULA). It’s just like when you need to check the box in the installation on the windows saying “Do you agree with the bla bla”.

To do that type:

sudo nano eula.txt

Rewrite “eula=false” to “eula=true” and save it be pressing “ctrl+o” shortcut. After that use “ctrl + x” shortcut to leave an editor.

Use upper arrow key or type again:

sudo java -Xms512M -Xmx1024M -jar /home/pi/minecraft_server/spigot-YourVersion.jar nogui

Give it a 10 or so minutes, the server needs to build a new map, prepare a spawn area etc.

When it says Done on the last line your server is up and running. Congratulation!

5. Connecting to your Minecraft server from a local network

Unfortunately for now you can only connect to the server from your local network at home. I’ll explain how to make server public in another step.

Your minecraft server is running on your Raspberry Pi IP address. If you don’t know what IP address do you have, you can find out by opening a new terminal window and typing the following command:

sudo hostname -I

Now start your minecraft launcher on your PC and click Join game>Add Server and insert your IP address.

6. Connecting to your Minecraft server from a public network

I’m gonna show you a two methods how to do this. First will be Port Forwarding method which should be better overall but it may not work for you (it didn’t work for me either) and second will be with the software called ngrok.

a) Port Forwardning Method

There is hunderd of tutorials on how to do this so I’m not gonna describe it in a detail. Here are some guides [1,2,3] or just google “how to portforward [NameOfYourRuter].

You will need to connect to your home router. For that just type 192.168.0.1 or 192.168.1.1 into your web browser and login in with your details.

I have tp-link router so I’ll be using screenshots from that. Go to Advanced>Nat Forwarding>Virtual Servers and click Add.

Type in following

Name into Service Type. I named mine MC Server.

External Port: 25565

Internal IP: IP address of your Raspberry PI (You are using this address to connect to your local MC server as well)

Internal Port: 25565

Protocol: All

And press the Save button.

If you don’t know what public IP address is or if you have one you probably don’t.

To find out your external IP (IP that other people and devices see you on the internet) just type “what is my ip into a google”

Keep in mind that this IP address can change every few days or whenever you reboot your router so if your minecraft server connection stop working it’s probably because of that.

Your other choice is to call your ISP provider and buy a public IP from him, but that can be pretty expensive sometimes.

Now insert this IP address to your minecraft multiplayer with a port number :25565

It’s going to look something like this

Your server should now be publicly visible. If you did everything as I did and It’s still not working don’t worry I have another method.

b) Using ngrok to make your MC server public

Sometimes your ISP provider is blocking some of the ports or is using NAT in their network. If that’s the case you can’t just port forward your external IP address. You can either buy public IP or use VPN service like ngrok, that allow us to bypass this.

You can read all about ngrok on their site. But basically ngrok serve as a middle man between our router and people on the other side. But our ISP provider doesn’t see what’s going on in this tunnel and it cannot be blocked. It also bypasses NAT so if your provider is using that you should have no problem now. If you are using VPN it’s kinda like that.

First you need to register here.

After your registration, you need to confirm a new account in your email address. When it’s done we can continue by download their software into our Raspberry PI.

To do that, open up a terminal and type:

cd /home/pi/
mkdir ngrok
cd ngrok
wget https://bin.equinox.io/c/4VmDzA7iaHb/ngrok-stable-linux-arm.zip

Now to unzip it type:

unzip ngrok-stable-linux-arm.zip

After you have successfully finished your registration, you should see a site with an authentication token. It’s DIFFERENT for everybody and it’s made from your account and password. It should look like this

Now copy the whole command with your token and paste it into a terminal in your Raspbery PI

./ngrok authtoken YourToken

After that you are ready to start ngrok. To do that, type command bellow, but instead of YourRegion type your actually region shortcut (you can find all of them bellow command)

./ngrok tcp -region= YOURREGION 25565

Because I’m from EU my command looks like this

./ngrok tcp -region eu 25565

Your ngrok should now be up and running.

Copy your IP address (one in the red rectangle) and paste it into your MC multiplayer Add server.

Congratulation, your Server is now public for all of your friends! (As long as ngrok is running of course).
