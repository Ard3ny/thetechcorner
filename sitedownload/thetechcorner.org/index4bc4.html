You’ve probably done this task many times before on your PC or phone, but in Linux everything is done over command, so let’s get right into it.

List password information about the user

chage -l username

Change the password

As a simple user without the privileges you can only change your own password but as a root you can change every user password including the root.

passwd username

Adjust password aging

Set the maximum number of days during which a password is valid (90 days for example)

chage -M 90 username

Set the minimum number of days between password changes (30 days for example)

chage -m 30 username

To make user immediately change the password after login

chage -d 0 username

Set expire date

chage -E EXPIRE_DATE username

A firewall is a set of rules. When a data packet moves into or out of a protected network space, its contents (in particular, information about its origin, target, and the protocol it plans to use) are tested against the firewall rules to see if it should be allowed through

We can manage firewall with firewall-cmd administrative tool.

Before I start talking about the actual commands, it would be good to know a few things first

Zones

The firewalld daemon manages groups of rules using entities called “zones”. Zones are basically sets of rules dictating what traffic should be allowed depending on the level of trust you have in the networks your computer is connected to. You may have strict rules in place prohibiting most traffic when operating on a public WiFi network, while allowing more relaxed restrictions when connected to your home network. 

Regardless of how dynamic your network environment may be, it is still useful to be familiar with the general idea behind each of the predefined zones for firewalld. In order from least trusted to most trusted, the predefined zones within firewalld are:

  • drop: The lowest level of trust. All incoming connections are dropped without reply and only outgoing connections are possible.
  • block: Similar to the above, but instead of simply dropping connections, incoming requests are rejected with an icmp-host-prohibited or icmp6-adm-prohibited message.
  • public: Represents public, untrusted networks. You don’t trust other computers but may allow selected incoming connections on a case-by-case basis.
  • external: External networks in the event that you are using the firewall as your gateway. It is configured for NAT masquerading so that your internal network remains private but reachable.
  • internal: The other side of the external zone, used for the internal portion of a gateway. The computers are fairly trustworthy and some additional services are available.
  • dmz: Used for computers located in a DMZ (isolated computers that will not have access to the rest of your network). Only certain incoming connections are allowed.
  • work: Used for work machines. Trust most of the computers in the network. A few more services might be allowed.
  • home: A home environment. It generally implies that you trust most of the other computers and that a few more services will be accepted.
  • trusted: Trust all of the machines in the network. The most open of the available options and should be used sparingly.

Rule Permanence

In firewalld, rules can be designated as either permanent or immediate. If a rule is added or modified, by default, the behavior of the currently running firewall is modified. At the next boot, the modifications will be thrown out and the old rules will be applied.

Most firewall-cmd operations can take the --permanent flag to indicate that the non-ephemeral firewall should be targeted. This will affect the rule set that is reloaded upon boot

Working with firewall

To get a zone, which is currently running on our system

firewall-cmd --get-active-zone

To get a zone, which is currently default on our system

firewall-cmd --get-default-zone

To get all the information about current active zone

firewall-cmd --list-all --zone=public

You can notice a lot of information, but I’m going to point out the service category, which is listing all currently allowed service traffic.

Removing the service from firewall whitelist

To remove service or rather I say to restrict the traffic we can simply remove the service

firewall-cmd --remove-service=nfs --pernament

To apply the change, we need to reload the settings or service

firewall-cmd --reload

Adding the service to firewall whitelist

firewall-cmd --add-service=nfs --pernament
firewall-cmd --reload

Changing the firewall zone

The default firewall zone we currently have is the “public”, but we can change this by typing

firewall-cmd --change-interface=*inteface_name* --zone=home

Changing the firewall default zone

If all of your interfaces can be handled well by a single zone, it’s probably easiest to just designate the best zone as default and then use that for your configuration.

You can change the default zone with the --set-default-zone= parameter. This will immediately change any interface using the default zone:

firewall-cmd --set-default-zone=home
]]> https://thetechcorner.org/?feed=rss2&p=1250 1 1250 How to Change Raspberry PI’s Hostname / Password https://thetechcorner.org/?p=177&utm_source=rss&utm_medium=rss&utm_campaign=how-to-change-raspberry-pis-hostname-password Wed, 06 Jan 2021 10:59:49 +0000 https://thetechcorner.org/?p=177 By default your Raspberry PI’s hostname is raspberrypi and your password is raspberry.

There is nothing wrong with this name until you have multiple PIes in your network and trust me when you buy one, you will want more of them to do even more cool projects.

So in this short guide I’ll show you how to change the hostname and the password.

We can choose multiple paths to do so. In this guide I’ll show you one way over CLI (console) and one way over GUI (graphical interface)

Changing Hostname / Password Using CLI

Because our PI is running Raspbian we can use one of it’s features that is called raspi-config which allow us to change these things with very friendly user interface.

Connect and login to your PI, you can find how in this guide.

In the console type:

sudo raspi-config

You should see option menu like this. Choose the System option.

Select the S4 Hostname and choose a new hostname. Keep in mind that you can only use ASCII letters a-z, the digits 0-9 and the hyphen, but hostname cannot begin nor end with hyphen.

Congratulation, now do the same for the password and you will be send into a terminal in which you will type your new password.

Keep in mind that you need to reboot your device before hostname changes will take a place. For reboot just type:

sudo shutdown -r +0

Changing Hostname / Password Using GUI

If you have installed Raspberry PI OS with desktop and you are more used to work with graphical intreface, this is even easier method of changing the hostname or password.

First you need to connect your monitor or launch VNC to connect to the PI with ahernet cable. You can find guide for that here.

In the left upper corner select Raspbian Icon and select Prefrences>Raspberry Pi Configuration.

You should see an option to change hostname and password. Click into the window where it says raspberrypi and rename it to whatever you want.

Then click on Change Password and pick your new password.

Congratulation, you have successfully changed your hostname and password. Now you dont have worry about logging into a wrong PI or about somebody logging into yours!
